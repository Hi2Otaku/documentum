1. **get_inbox_items(db, user_id, skip, limit, state_filter, priority_filter, template_name_filter, sort_by, sort_order)** -> tuple[list[dict], int]
   - Per D-01/D-02: Query WorkItem where performer_id == user_id AND is_deleted == False
   - Per D-03: Apply optional filters:
     - state_filter: WorkItem.state == state_filter (e.g. "available", "acquired")
     - priority_filter: WorkItem.priority == priority_filter (int)
     - template_name_filter: join through ActivityInstance -> WorkflowInstance -> ProcessTemplate WHERE ProcessTemplate.name.ilike(f"%{template_name_filter}%")
   - Per D-03: Sort by sort_by field (default "created_at"), sort_order "asc" or "desc" (default "desc")
     - Valid sort fields: "priority", "due_date", "created_at"
   - Use selectinload chains for eager loading (per Research Pattern 2):
     - selectinload(WorkItem.activity_instance).selectinload(ActivityInstance.activity_template)
     - selectinload(WorkItem.activity_instance).selectinload(ActivityInstance.workflow_instance).selectinload(WorkflowInstance.process_template)
     - selectinload(WorkItem.activity_instance).selectinload(ActivityInstance.workflow_instance).selectinload(WorkflowInstance.workflow_packages)
     - selectinload(WorkItem.comments)
   - Build InboxItemResponse-compatible dicts manually in the service (not relying on from_attributes for deep nesting):
     ```python
     items = []
     for wi in work_items:
         ai = wi.activity_instance
         at = ai.activity_template
         wf = ai.workflow_instance
         pt = wf.process_template
         items.append({
             "id": wi.id, "state": wi.state, "priority": wi.priority,
             "due_date": wi.due_date, "instructions": wi.instructions,
             "performer_id": wi.performer_id, "created_at": wi.created_at,
             "completed_at": wi.completed_at,
             "activity": {"name": at.name, "activity_type": at.activity_type.value, "instructions": at.description},
             "workflow": {"id": wf.id, "template_name": pt.name, "state": wf.state},
             "documents": [{"document_id": p.document_id, "package_name": p.package_name} for p in wf.workflow_packages],
             "comment_count": len(wi.comments),
         })
     ```
   - Return (items, total_count) for pagination. Use a separate COUNT query with the same WHERE clause.

2. **get_inbox_item_detail(db, work_item_id, user_id)** -> dict
   - Load single WorkItem by id with same selectinload chains as above
   - Verify performer_id == user_id (raise ValueError "Not authorized to view this work item" otherwise)
   - Build dict like above but include "comments" list with CommentResponse-compatible dicts
   - Per INBOX-03: Shows documents and activity details

3. **acquire_work_item(db, work_item_id, user_id)** -> WorkItem
   - Per D-08: Load WorkItem with with_for_update() (SELECT ... FOR UPDATE to prevent race condition per Research Pitfall 4)
   - Verify state is AVAILABLE, raise ValueError if not
   - Set state = ACQUIRED, performer_id = uuid.UUID(user_id)
   - Create audit record: entity_type="work_item", action="work_item_acquired"
   - Return work item

4. **release_work_item(db, work_item_id, user_id)** -> WorkItem
   - Per D-09: Load WorkItem, verify state is ACQUIRED and performer_id matches user_id
   - Set state = AVAILABLE, performer_id = None
   - Create audit record: action="work_item_released"
   - Return work item

5. **complete_inbox_item(db, work_item_id, user_id, output_variables)** -> WorkItem
   - Per D-08: Load WorkItem with selectinload(WorkItem.activity_instance), verify state is ACQUIRED and performer_id matches user_id
   - Get workflow_instance_id from work_item.activity_instance.workflow_instance_id
   - Delegate to engine_service.complete_work_item(db, workflow_instance_id, work_item_id, user_id, output_variables)
   - Per INBOX-04: This advances the workflow. Do NOT reimplement advancement logic.

6. **reject_inbox_item(db, work_item_id, user_id, reason)** -> WorkItem
   - Per D-04: Load WorkItem, verify state is ACQUIRED and performer_id matches user_id
   - Set state = REJECTED, completed_at = datetime.now(timezone.utc)
   - Create audit record: action="work_item_rejected", after_state includes reason if provided
   - Per D-04: Does NOT trigger reject flow routing (Phase 6)
   - Return work item

7. **add_comment(db, work_item_id, user_id, content)** -> WorkItemComment
   - Per INBOX-06/D-05: Verify work item exists and is not deleted
   - Create WorkItemComment(work_item_id=work_item_id, user_id=uuid.UUID(user_id), content=content, created_by=user_id)
   - Create audit record: action="work_item_comment_added"
   - Return the comment

8. **get_comments(db, work_item_id)** -> list[WorkItemComment]
   - Query WorkItemComment where work_item_id matches, ordered by created_at ASC

Import from engine_service only what's needed: complete_work_item. Import WORK_ITEM_TRANSITIONS for state validation if desired, or just check states directly.
