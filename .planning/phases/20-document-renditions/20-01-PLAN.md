# Phase 20 Plan 01: Rendition Model, Celery Tasks, and API

## Objective
Add document rendition support: a Rendition database model tracking PDF/thumbnail renditions per document version, Celery tasks that generate renditions (PDF via LibreOffice headless, thumbnails via Pillow), API endpoints to list/download/retry renditions, and automatic triggering on document upload/checkin.

## Context
- @src/app/models/document.py - Document and DocumentVersion models
- @src/app/services/document_service.py - upload/checkin flows
- @src/app/routers/documents.py - document API endpoints
- @src/app/celery_app.py - Celery config
- @src/app/core/minio_client.py - MinIO storage helpers

## Tasks

### Task 1: Rendition Model and Migration
type="auto"

Create the `Rendition` SQLAlchemy model and Alembic migration.

**Actions:**
- Create `src/app/models/rendition.py` with Rendition model (id, document_version_id FK, rendition_type enum [PDF, THUMBNAIL], status enum [PENDING, PROCESSING, READY, FAILED], minio_object_key, content_type, content_size, error_message, created_at/updated_at)
- Add RenditionType and RenditionStatus enums to `src/app/models/enums.py`
- Register model in `src/app/models/__init__.py`
- Create Alembic migration

**Done criteria:** Model file exists with proper relationships, migration file created.

### Task 2: Rendition Celery Tasks
type="auto"

Create Celery tasks for PDF and thumbnail generation.

**Actions:**
- Create `src/app/tasks/rendition.py` with:
  - `generate_pdf_rendition(document_version_id, document_id)` - uses subprocess to call LibreOffice headless (or marks as failed if not available), uploads result to MinIO
  - `generate_thumbnail(document_version_id, document_id)` - uses Pillow for images or a placeholder for non-image types, uploads to MinIO
  - Both tasks update Rendition status in DB (PROCESSING -> READY or FAILED)
- Register tasks in `src/app/celery_app.py` include list
- Use synchronous SQLAlchemy session (Celery worker context) for DB access

**Done criteria:** Task file exists, registered in Celery app config.

### Task 3: Rendition Service and API Endpoints
type="auto"

Create service layer and REST API for renditions.

**Actions:**
- Create `src/app/services/rendition_service.py` with:
  - `create_rendition_request(db, version_id, rendition_type)` - creates PENDING record and dispatches Celery task
  - `get_renditions_for_version(db, document_id, version_id)` - list renditions
  - `get_rendition(db, rendition_id)` - get single rendition
  - `retry_rendition(db, rendition_id)` - reset FAILED to PENDING and re-dispatch
  - `download_rendition(db, rendition_id)` - download from MinIO
- Create `src/app/schemas/rendition.py` with response schemas
- Create `src/app/routers/renditions.py` with endpoints:
  - GET `/documents/{document_id}/versions/{version_id}/renditions` - list renditions
  - GET `/renditions/{rendition_id}/download` - download rendition
  - POST `/renditions/{rendition_id}/retry` - retry failed rendition
- Register router in `src/app/main.py`

**Done criteria:** All endpoints registered, service functions created.

### Task 4: Auto-trigger Renditions on Upload/Checkin
type="auto"

Wire rendition generation into document upload and checkin flows.

**Actions:**
- Modify `src/app/services/document_service.py` upload_document and checkin_document to dispatch rendition tasks after successful creation
- Create both PDF and THUMBNAIL rendition requests for each new version
- Import and call rendition_service functions

**Done criteria:** Uploading or checking in a document automatically creates PENDING rendition records and dispatches Celery tasks.

## Verification
