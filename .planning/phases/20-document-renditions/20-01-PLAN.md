    1. Create `src/app/services/rendition_service.py` with these functions:

       a. `CONVERTIBLE_TYPES` set — office MIME types that LibreOffice can convert: application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document, .spreadsheetml.sheet, .presentationml.presentation, vnd.oasis.opendocument.text, .opendocument.spreadsheet, text/plain, text/csv, text/html

       b. `IMAGE_TYPES` set — image/jpeg, image/png, image/gif, image/webp, image/tiff, image/bmp

       c. `PDF_TYPE = "application/pdf"`

       d. `MAX_FILE_SIZE = 50 * 1024 * 1024` (50MB limit per research pitfall 4)

       e. `THUMB_SIZE = (200, 200)`

       f. `async def create_pending_renditions(db: AsyncSession, version: DocumentVersion) -> tuple[Rendition, Rendition]`:
          - Create two Rendition rows (PDF + THUMBNAIL) with status=PENDING
          - db.add_all + flush
          - Return the pair

       g. `def convert_to_pdf(source_bytes: bytes, filename: str) -> bytes`:
          - Use tempfile.TemporaryDirectory
          - Write source to temp dir, run `soffice --headless --convert-to pdf --outdir {tmpdir} -env:UserInstallation=file:///tmp/libreoffice_profile_{uuid4()} {src_path}` (unique profile per research pitfall 1)
          - On Windows, try `soffice` then fall back to checking common paths; on FileNotFoundError raise RuntimeError("LibreOffice not available")
          - timeout=120 on subprocess.run, capture_output=True
          - Read and return resulting PDF bytes
          - On non-zero return code, raise RuntimeError with stderr

       h. `def generate_thumbnail_from_pdf(pdf_bytes: bytes) -> bytes`:
          - Import pypdfium2, open PDF, render page 0 at scale=1
          - Convert to PIL, thumbnail to THUMB_SIZE with LANCZOS
          - Save to BytesIO as PNG, return bytes

       i. `def generate_thumbnail_from_image(image_bytes: bytes) -> bytes`:
          - Open with Pillow, thumbnail to THUMB_SIZE with LANCZOS
          - Save as PNG, return bytes

       j. `def can_generate_pdf(content_type: str) -> bool`: returns True if content_type in CONVERTIBLE_TYPES or content_type == PDF_TYPE

       k. `def can_generate_thumbnail(content_type: str) -> bool`: returns True if can_generate_pdf(content_type) or content_type in IMAGE_TYPES

    2. Create `src/app/tasks/rendition.py` with two Celery tasks following the notification.py pattern:

       a. `@celery_app.task(name="app.tasks.rendition.generate_pdf_rendition", bind=True, max_retries=2, default_retry_delay=30)`
          `def generate_pdf_rendition(self, version_id: str, rendition_id: str):`
          - Calls `asyncio.run(_generate_pdf_async(self, version_id, rendition_id))`

       b. `async def _generate_pdf_async(task, version_id, rendition_id):`
          - Use `create_task_session_factory()` pattern
          - Load DocumentVersion and Rendition by ID
          - Check content_size against MAX_FILE_SIZE; if too large, set FAILED with message "File too large for rendition"
          - Download source from MinIO using `minio_client.download_object(version.minio_object_key)` (sync, use `minio_client.minio_client.get_object` in thread)
          - If content_type is PDF: source bytes ARE the PDF (pass-through)
          - If content_type in CONVERTIBLE_TYPES: call `convert_to_pdf(source_bytes, version.filename)`
          - Otherwise: set FAILED "Unsupported content type for PDF rendition"
          - Upload PDF bytes to MinIO at key `{version.document_id}/{version_id}/renditions/pdf.pdf` with content_type="application/pdf"
          - Update rendition: status=READY, minio_object_key, content_type="application/pdf", content_size=len(pdf_bytes)
          - On any exception: set status=FAILED, error_message=str(e), commit

       c. `@celery_app.task(name="app.tasks.rendition.generate_thumbnail", bind=True, max_retries=2, default_retry_delay=30)`
          `def generate_thumbnail(self, version_id: str, rendition_id: str):`
          - Calls `asyncio.run(_generate_thumbnail_async(self, version_id, rendition_id))`

       d. `async def _generate_thumbnail_async(task, version_id, rendition_id):`
          - Same session factory pattern
          - Load version + rendition
          - If content_type in IMAGE_TYPES: download source, call `generate_thumbnail_from_image`
          - If content_type is PDF or in CONVERTIBLE_TYPES: 
            - For PDF: download, call `generate_thumbnail_from_pdf`
            - For office types: first convert to PDF, then call `generate_thumbnail_from_pdf`
          - Otherwise: set FAILED "Unsupported content type for thumbnail"
          - Upload thumbnail to MinIO at `{doc_id}/{version_id}/renditions/thumbnail.png` with content_type="image/png"
          - Update rendition: status=READY, minio_object_key, content_type="image/png", content_size
          - On exception: FAILED with error_message

    3. Add to `src/app/services/event_handlers.py`:
       - Import `rendition_service` and Rendition/RenditionType/RenditionStatus from models
       - Add handler `@event_bus.on("document.uploaded")` named `_trigger_renditions_on_upload(db, event)`:
         - Get entity_id (document_id) from event

         - Otherwise: immediately set thumb rendition status=FAILED, error_message="Unsupported format"
         - await db.flush()
