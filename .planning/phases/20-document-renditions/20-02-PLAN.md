       - Response: `EnvelopeResponse[list[RenditionResponse]]`
       - Query: `select(Rendition).where(Rendition.document_version_id == version_id)` 
       - Verify the version belongs to the document (join or subquery check)
       - Return all renditions for that version
       - Requires authenticated user (get_current_user dependency)

    2. **GET /documents/{document_id}/versions/{version_id}/renditions/{rendition_id}/download**
       - Load the Rendition by ID, verify it belongs to the correct version
       - If status != READY: return 404 with detail "Rendition not ready"
       - If minio_object_key is None: return 404
       - Download bytes from MinIO via `download_object(rendition.minio_object_key)`
       - Return `Response(content=data, media_type=rendition.content_type, headers={"Content-Disposition": f"attachment; filename=rendition_{rendition.rendition_type}.{ext}"})`
         - ext: "pdf" for PDF type, "png" for THUMBNAIL type
       - Requires authenticated user

    3. **POST /documents/{document_id}/versions/{version_id}/renditions/{rendition_id}/retry**
       - Response: `EnvelopeResponse[RenditionResponse]`
       - Load Rendition, verify it belongs to correct version
       - If status != FAILED: return 400 "Only failed renditions can be retried"
       - Reset status to PENDING, clear error_message, flush

         - PDF: `generate_pdf_rendition.delay(str(version_id), str(rendition_id))`
         - THUMBNAIL: `generate_thumbnail.delay(str(version_id), str(rendition_id))`
       - Return updated rendition

  <action>
    Add a `celery-rendition-worker` service to docker-compose.yml:

    ```yaml
    celery-rendition-worker:
      build: .
      environment:
        - DATABASE_URL=postgresql+asyncpg://docuser:docpass@db:5432/documentum
        - REDIS_URL=redis://redis:6379/0
        - SECRET_KEY=change-me-in-production
        - MINIO_ENDPOINT=minio:9000
        - MINIO_ACCESS_KEY=minioadmin
        - MINIO_SECRET_KEY=minioadmin
      depends_on:
        db:
          condition: service_healthy
        redis:
          condition: service_healthy
        minio:
          condition: service_started
      command: celery -A app.celery_app worker --loglevel=info -Q renditions --concurrency=1
    ```

    Key details:
    - Uses `-Q renditions` to only consume from the renditions queue (isolates heavy conversion work)
    - Uses `--concurrency=1` to prevent LibreOffice global lock issues (research pitfall 1)
    - Includes MINIO_* env vars so the worker can access document storage
    - Copy environment pattern from existing celery-worker service, adding MinIO vars

    Also update the existing `celery-worker` service command to explicitly exclude the renditions queue by NOT changing it (default queue handles everything else). The rendition tasks will be routed to the "renditions" queue.

    Update `src/app/celery_app.py` to add task routing:
    ```python
    celery_app.conf.update(
        ...
        task_routes={
            "app.tasks.rendition.*": {"queue": "renditions"},
        },
    )
    ```
    (This ensures rendition tasks go to the dedicated worker queue.)
