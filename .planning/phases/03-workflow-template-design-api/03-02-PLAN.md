**Helper — get template with all relations (used everywhere):**
```python
async def get_template_with_relations(db: AsyncSession, template_id: UUID) -> ProcessTemplate | None:
    result = await db.execute(
        select(ProcessTemplate)
        .options(
            selectinload(ProcessTemplate.activity_templates),
            selectinload(ProcessTemplate.flow_templates),
            selectinload(ProcessTemplate.process_variables),
        )
        .where(ProcessTemplate.id == template_id, ProcessTemplate.is_deleted == False)
    )
    return result.scalar_one_or_none()
```

**Helper — reset state to Draft if template is Validated:**
```python
async def _reset_to_draft_if_validated(db: AsyncSession, template: ProcessTemplate):
    if template.state == ProcessState.VALIDATED:
        template.state = ProcessState.DRAFT
```

**ProcessTemplate CRUD:**

`create_template(db, data: ProcessTemplateCreate, user_id: str) -> ProcessTemplate`:
- Create ProcessTemplate with name, description, version=1, state=DRAFT, is_installed=False, created_by=user_id
- Audit: entity_type="process_template", action="create"

`get_template(db, template_id: UUID) -> ProcessTemplate | None`:
- Call get_template_with_relations

`list_templates(db, page: int = 1, page_size: int = 20) -> tuple[list[ProcessTemplate], int]`:
- Query ProcessTemplate where is_deleted=False, order by created_at desc
- Return (list, total_count) for pagination
- Use selectinload for all three relationships

`update_template(db, template_id: UUID, data: ProcessTemplateUpdate, user_id: str) -> ProcessTemplate`:
- Fetch template, raise ValueError if not found or is_deleted
- If template.state == ProcessState.ACTIVE (installed), call `create_new_version` instead and update the NEW version
- If template.state == ProcessState.VALIDATED, reset to DRAFT
- Apply non-None fields from data
- Audit: action="update"

`delete_template(db, template_id: UUID, user_id: str) -> ProcessTemplate`:
- Soft delete (set is_deleted=True)
- Audit: action="delete"

**ActivityTemplate CRUD:**

`add_activity(db, template_id: UUID, data: ActivityTemplateCreate, user_id: str) -> ActivityTemplate`:
- Fetch parent template, verify exists and not deleted
- If parent is ACTIVE, raise ValueError("Cannot modify installed template. Create a new version first.")
- Reset state to Draft if Validated
- Create ActivityTemplate with process_template_id=template_id, all fields from data
- For performer_type, store the string value of the PerformerType enum (e.g., "user", "group")
- Audit: entity_type="activity_template", action="create"

`update_activity(db, template_id: UUID, activity_id: UUID, data: ActivityTemplateUpdate, user_id: str) -> ActivityTemplate`:
- Fetch activity, verify it belongs to template_id, not deleted
- Parent template same checks as add_activity (no modify if ACTIVE, reset if VALIDATED)
- Apply non-None fields
- Audit: action="update"

`delete_activity(db, template_id: UUID, activity_id: UUID, user_id: str) -> ActivityTemplate`:
- Soft delete activity, verify belongs to template
- Parent template state check (no modify if ACTIVE, reset if VALIDATED)
- Audit: action="delete"

**FlowTemplate CRUD:**

`add_flow(db, template_id: UUID, data: FlowTemplateCreate, user_id: str) -> FlowTemplate`:
- Verify both source_activity_id and target_activity_id belong to this template and are not deleted
- Verify source != target (no self-loops)
- If data.condition_expression is a dict, serialize to JSON string: `json.dumps(data.condition_expression)`
- Parent template state check (no modify if ACTIVE, reset if VALIDATED)
- Audit: entity_type="flow_template", action="create"

`update_flow(db, template_id: UUID, flow_id: UUID, data: FlowTemplateUpdate, user_id: str) -> FlowTemplate`:
- Verify flow belongs to template
- Parent template state check
- If data.condition_expression is a dict, serialize to JSON string
- Audit: action="update"

`delete_flow(db, template_id: UUID, flow_id: UUID, user_id: str) -> FlowTemplate`:
- Soft delete, verify belongs to template
- Parent template state check
- Audit: action="delete"

**ProcessVariable CRUD:**

**Validation (TMPL-08):**

`validate_template(db, template_id: UUID, user_id: str) -> tuple[bool, list[dict]]`:
- Fetch template with relations
- Collect errors as list of dicts: `{"code": str, "message": str, "entity_type": str, "entity_id": str | None}`
- Filter out soft-deleted sub-entities for all checks

Validation checks (in order):
1. **INVALID_START_COUNT**: Exactly 1 activity with activity_type == START
2. **NO_END_ACTIVITY**: At least 1 activity with activity_type == END
3. **UNREACHABLE_ACTIVITY**: BFS from Start — use a deque (from collections). Build adjacency from flows (source -> [targets]). Any non-visited activity is unreachable.
4. **ORPHAN_FLOW_IN**: Every non-Start activity must have >= 1 incoming flow
5. **ORPHAN_FLOW_OUT**: Every non-End activity must have >= 1 outgoing flow
6. **MISSING_PERFORMER**: Every MANUAL activity must have performer_type set (not None, not empty string)
7. **MISSING_METHOD**: Every AUTO activity must have method_name set (not None, not empty string)
8. **SELF_LOOP**: No flow where source_activity_id == target_activity_id
9. **INVALID_CONDITION**: If flow has condition_expression, try `json.loads()` if string; validate it has "field" and "operator" keys (or "all"/"any" for compound). Check that referenced field names match variable names on the template.

If errors is empty:
- Set template.state = ProcessState.VALIDATED
- Audit: entity_type="process_template", action="validate", details="Template validated successfully"
- Return (True, [])

If errors is not empty:
- Keep state as DRAFT
- Return (False, errors)

**Installation (TMPL-09):**

`install_template(db, template_id: UUID, user_id: str) -> ProcessTemplate`:
- Fetch template, verify state == VALIDATED (raise ValueError if not)
- Set state = ProcessState.ACTIVE, is_installed = True, installed_at = datetime.now(timezone.utc)
- Deprecate any other installed versions with the same name: query ProcessTemplate where name == template.name AND is_installed == True AND id != template_id, set state = DEPRECATED, is_installed = False
- Audit: action="install"

**Versioning — copy-on-write (TMPL-10, per D-02):**

`create_new_version(db, template_id: UUID, user_id: str) -> ProcessTemplate`:
- Fetch original template with relations (must be ACTIVE/installed)
- Create new ProcessTemplate: same name, description, version = original.version + 1, state = DRAFT, is_installed = False, created_by = user_id
- db.add(new_template), await db.flush() to get new_template.id
- Build activity_id_map: dict[UUID, UUID] mapping old activity IDs to new ones
- Clone each non-deleted ActivityTemplate: copy all fields, set process_template_id = new_template.id. db.add, flush, record mapping.
- Clone each non-deleted FlowTemplate: copy all fields, remap source_activity_id and target_activity_id using activity_id_map, set process_template_id = new_template.id
- Clone each non-deleted ProcessVariable: copy all fields, set process_template_id = new_template.id
- Audit: entity_type="process_template", action="create_version", details=f"New version {new_template.version} created from version {original.version}"
- Return new_template
