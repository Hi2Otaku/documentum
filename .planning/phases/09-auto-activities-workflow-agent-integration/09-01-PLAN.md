<objective>
Create the auto method registry, ActivityContext, four built-in auto methods, execution log model, and update the engine to handle AUTO activities.

Purpose: This plan establishes all the domain-level components for automated activity execution. The registry pattern (D-01) allows methods to be registered via decorator and looked up by the `method_name` field on ActivityTemplate. The engine must stop skipping AUTO activities and instead leave them in ACTIVE state for the Workflow Agent (Plan 02) to pick up.

Output: Auto method infrastructure ready for Celery integration.
</objective>

    - `@dataclass` class `ActivityContext` with fields:
      - `db: AsyncSession`
      - `workflow_instance: WorkflowInstance`
      - `activity_instance: ActivityInstance`
      - `activity_template: ActivityTemplate`
      - `variables: dict[str, Any]` (name -> resolved value)
      - `document_ids: list[uuid.UUID]` (from WorkflowPackage)
      - `user_id: str` (default "system" for auto activities)
    - `async def get_variable(self, name: str) -> Any` — returns `self.variables.get(name)`
    - `async def set_variable(self, name: str, value: Any) -> None` — updates in-memory dict AND updates the ProcessVariable row in DB via `self.db`. Use the `_set_variable_value` pattern from engine_service (match on variable_type). Query ProcessVariable by workflow_instance_id and name, then set the typed column.

    **`src/app/auto_methods/builtin.py`** (per D-03):
    Four methods, all async, all accept `ActivityContext`, all return `dict | None`:

    1. `@auto_method("send_email")` — reads `email_to`, `email_subject`, `email_body` from process variables via `ctx.get_variable()`. If `settings.smtp_host` is empty string, log the email details at INFO level and return `{"mode": "dev", "to": email_to, ...}`. If smtp_host is set, use `smtplib.SMTP` (sync, wrapped in `asyncio.to_thread`) to send. Return `{"sent": True, "to": email_to}`.

    2. `@auto_method("change_lifecycle_state")` — reads `target_lifecycle_state` from process variables. Lazy-imports `LifecycleState` from `app.models.enums` and `transition_lifecycle_state` from `app.services.lifecycle_service`. Iterates `ctx.document_ids`, calls `await transition_lifecycle_state(ctx.db, doc_id, target_state, ctx.user_id)` for each. Returns `{"transitions": [{"document_id": str(id), "new_state": str}]}`.

    3. `@auto_method("modify_acl")` — reads `acl_action` ("add" or "remove"), `acl_user_id`, `acl_permission` from process variables. Lazy-imports from `app.services.acl_service`. For each document in `ctx.document_ids`, calls `create_acl_entry` or `remove_acl_entry`. Returns `{"modified": count}`.

    4. `@auto_method("call_external_api")` — reads `api_url` and `api_payload_template` (optional JSON string) from process variables. Uses `httpx.AsyncClient` to POST to the URL with JSON body containing `{"workflow_id": str, "activity_id": str, "variables": ctx.variables}` merged with payload template if provided. Timeout from `api_timeout` variable or 30 seconds default. Stores response status and body in `api_response_status` and `api_response_body` variables via `ctx.set_variable()`. Returns `{"status": response.status_code, "url": api_url}`. Per D-10, this covers INTG-01.
