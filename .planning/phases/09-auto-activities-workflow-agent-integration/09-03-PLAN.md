    - `test_retry_failed_activity`: create an AUTO activity in ERROR state via direct DB manipulation, POST to retry endpoint, verify:

      - Activity state reset to ACTIVE
    - `test_skip_failed_activity`: create an AUTO activity in ERROR state, POST to skip endpoint, verify:
      - Response 200 with status="skipped"
      - Activity state is COMPLETE
      - AutoActivityLog entry with status="skipped"
      - Workflow advanced (check next activity state)
    - `test_retry_non_error_activity_fails`: try to retry an ACTIVE activity, verify 400 response
    - `test_skip_non_error_activity_fails`: try to skip an ACTIVE activity, verify 400 response
