   - `router = APIRouter(prefix="/notifications", tags=["notifications"])`
   - Import get_current_active_user from core.dependencies (NOT get_current_active_admin -- notifications are for all users)
   
   - `GET /` endpoint: list notifications for current user
     - Query params: limit (int, default 20, max 100), offset (int, default 0), unread_only (bool, default False)
     - Call notification_service.list_notifications(db, current_user.id, limit, offset, unread_only)
     - Call notification_service.get_unread_count(db, current_user.id)
     - Return `EnvelopeResponse[NotificationListResponse]` with notifications list and unread_count
   
   - `PATCH /{notification_id}/read` endpoint: mark single notification as read
     - Call notification_service.mark_read(db, notification_id, current_user.id)
     - If None returned, raise 404 "Notification not found"
     - Return `EnvelopeResponse[NotificationResponse]`
   
   - `PATCH /read-all` endpoint: mark all as read
     - IMPORTANT: register this route BEFORE `/{notification_id}/read` to avoid path conflicts (FastAPI matches routes in order)
     - Call notification_service.mark_all_read(db, current_user.id)
     - Return `EnvelopeResponse[dict]` with {"updated_count": count}
   
   - `GET /stream` SSE endpoint: real-time notification stream
     - Query param: token (str, required) -- JWT for SSE auth (EventSource cannot send headers)
     - Validate token using same pattern as dashboard._validate_sse_token but WITHOUT requiring is_superuser (notifications are for all users)
     - Create `_validate_notification_sse_token(token: str) -> User` that validates JWT and returns any active user
     - Create `_notification_sse_generator(user_id: uuid.UUID)` async generator:
       - Get initial unread count from DB, yield `event: unread_count\ndata: {"count": N}\n\n`
       - Subscribe to Redis "notifications" pub/sub channel
       - In try/finally: listen for messages, filter by user_id match, yield `event: new_notification\ndata: {...}\n\n`
       - Send heartbeat `:heartbeat\n\n` every 30 seconds using asyncio.wait_for with timeout
       - In finally: unsubscribe and close pubsub
     - Return StreamingResponse with media_type="text/event-stream" and headers {"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}
