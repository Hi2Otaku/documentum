**Create `tests/test_routing.py`** with these tests (all async, use `client` and `auth_headers` fixtures from conftest):

**Helper function** `_create_routing_template(client, auth_headers, routing_type, num_paths=2)`:
Creates a template with: Start -> ManualActivity(routing_type=routing_type) -> End1, End2 (or more).
- POST /api/v1/templates to create template
- POST activities: start, manual (with `routing_type` field set), end1, end2
- POST flows: start->manual (NORMAL), manual->end1 (NORMAL, display_label="path_a", condition_expression="approved == True" for conditional), manual->end2 (NORMAL, display_label="path_b", condition_expression="approved == False" for conditional)
- POST /api/v1/templates/{id}/install
- Return template_id and activity/flow IDs dict

For performer-chosen template, set manual activity routing_type="performer_chosen". Flows get display_label="path_a" and "path_b" but no condition_expression.

For broadcast template, set manual activity routing_type="broadcast". Both flows have no conditions.

For conditional template, routing_type="conditional" (or omit for default). Flows have condition_expressions.

**Test: `test_performer_chosen_routing` (EXEC-08)**
1. Create performer-chosen template using helper
2. Start workflow: POST /api/v1/workflows
3. Get the work item from workflow detail: GET /api/v1/workflows/{id}
4. Acquire work item: POST /api/v1/inbox/{wi_id}/acquire
5. Complete with `selected_path="path_a"`: POST /api/v1/inbox/{wi_id}/complete with body `{"selected_path": "path_a"}`
6. Verify: GET workflow detail, assert workflow state is "finished", assert only the end activity connected to path_a completed (check activity_instances states)

**Test: `test_performer_chosen_missing_path_errors`**
Same setup, but complete WITHOUT selected_path. Expect 400 error with "requires selected_path".

**Test: `test_performer_chosen_invalid_path_errors`**
Complete with `selected_path="nonexistent"`. Expect 400 error with "No flow with label".

**Test: `test_conditional_routing` (EXEC-09)**
1. Create conditional template (default routing_type) with variable `approved` (boolean), two flows with conditions `approved == True` and `approved == False`
2. Start workflow with `initial_variables={"approved": true}`
3. Acquire and complete the manual activity (no selected_path needed)
4. Verify: only the path_a (approved==True) end activity completed

**Test: `test_broadcast_routing` (EXEC-10)**

2. Template: Start -> Manual(broadcast) -> EndA, EndB (two end activities, both via NORMAL flow)
3. Start workflow, acquire and complete manual activity
4. Verify: BOTH end activities completed, workflow finished
