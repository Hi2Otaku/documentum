**B. Modify `_advance_from_activity` for routing type dispatch (per D-02):**
In the `while queue:` loop, after getting `outgoing_flows` (NORMAL flows), look up the current activity template's `routing_type` from `activity_template_map`:

```python
current_at = activity_template_map.get(current.activity_template_id)
routing_type = getattr(current_at, 'routing_type', None) or "conditional"

match routing_type:
    case "broadcast":
        flows_to_fire = outgoing_flows  # Fire ALL, ignore conditions
    case "performer_chosen":
        if not selected_path:
            raise ValueError("Performer-chosen activity requires selected_path")
        flows_to_fire = [f for f in outgoing_flows if f.display_label == selected_path]
        if not flows_to_fire:
            raise ValueError(f"No flow with label '{selected_path}'")
    case "conditional" | _:
        # Existing behavior: evaluate condition_expression
        flows_to_fire = []
        for flow in outgoing_flows:
            if flow.condition_expression:
                try:
                    if not evaluate_expression(flow.condition_expression, var_context):
                        continue
                except Exception:
                    logger.warning("Condition evaluation failed for flow %s, skipping", flow.id)
                    continue
            flows_to_fire.append(flow)
```

Replace the existing per-flow condition evaluation loop with this dispatch. The `selected_path` parameter must be added to `_advance_from_activity` signature: `selected_path: str | None = None`.

**IMPORTANT:** The `selected_path` should only be used for the FIRST activity in the queue (the one the user completed). For auto-completing activities (start/end) in the queue, `selected_path` should be None. Add a flag or only pass `selected_path` when processing the first item. Simplest approach: after the first iteration, set `selected_path = None` so subsequent queue items use default routing.

```python

    reason: str | None = None,
) -> WorkItem:
```
Steps:
1. Load workflow instance, verify RUNNING state
2. Load work item with selectinload(activity_instance), verify state is ACQUIRED
3. Mark work item state = REJECTED, completed_at = now
4. Load template with selectinload (activity_templates, flow_templates)

   ```python

   ```

7. Build template_to_instance mapping (query all ActivityInstances for this workflow)
8. For each reject flow:
   - Get target activity instance from template_to_instance
   - Reset target: `_enforce_activity_transition(target_ai.state, ActivityState.ACTIVE)` (works because we added COMPLETE->ACTIVE)
   - Set `target_ai.state = ActivityState.ACTIVE`, `target_ai.started_at = datetime.now(UTC)`, `target_ai.completed_at = None`
   - Resolve performers for the target activity template (using resolve_performers)
   - Create new work items for each resolved performer (old work items remain in history per D-04)
   - Place ExecutionToken for the reject flow (workflow_instance_id, flow_template_id, source=current activity instance, target=reject flow target template)
9. Create audit record: action="work_item_rejected", include reason if provided
10. Flush and return work item

```python
async def reject_inbox_item(
    db: AsyncSession,
    work_item_id: uuid.UUID,
    user_id: str,
    reason: str | None = None,
) -> WorkItem:
    from app.services import engine_service
    # Load work item to get workflow_instance_id
    result = await db.execute(
        select(WorkItem)
        .options(selectinload(WorkItem.activity_instance))
        .where(WorkItem.id == work_item_id, WorkItem.is_deleted == False)
    )
    wi = result.scalar_one_or_none()
    if wi is None:
        raise ValueError("Work item not found")
    if wi.state != WorkItemState.ACQUIRED:
        raise ValueError(f"Cannot reject work item in state '{wi.state.value}'; must be 'acquired'")
    if wi.performer_id != uuid.UUID(user_id):
        raise ValueError("Not authorized to reject this work item")

    workflow_instance_id = wi.activity_instance.workflow_instance_id
    return await engine_service.reject_work_item(
        db, workflow_instance_id, work_item_id, user_id, reason
    )
```
