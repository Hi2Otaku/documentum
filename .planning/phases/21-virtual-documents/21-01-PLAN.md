  <action>
    Create `src/app/services/virtual_document_service.py` following the established service pattern (async functions, AsyncSession, HTTPException for errors, audit records).

    **Functions to implement:**

    1. `async def create_virtual_document(db, title, description, user_id) -> VirtualDocument`:
       - Create a Document record with title, author=None, filename=f"{title}.vdoc", content_type="application/x-virtual-document"
       - Create a VirtualDocument record pointing to that Document
       - Create audit record (entity_type="virtual_document", action="create")
       - Return the VirtualDocument with eagerly loaded document relationship

    2. `async def get_virtual_document(db, virtual_doc_id) -> VirtualDocument`:
       - Select VirtualDocument by id, is_deleted=False
       - Eagerly load children (with child_document relationship) and document
       - Raise 404 if not found
       - Return with children sorted by order_index

    3. `async def list_virtual_documents(db, page, page_size) -> tuple[list, int]`:
       - Paginated list of virtual documents with total count
       - Eagerly load document relationship for title access

    4. `async def add_child(db, virtual_doc_id, child_document_id, user_id) -> VirtualDocumentChild`:
       - Get virtual document (404 if missing)
       - Verify child document exists and is not deleted (404 if missing)
       - **Cycle detection**: Call `_detect_cycle(db, virtual_doc_id, child_document_id)` -- if True, raise 400 "Circular reference detected: adding this document would create a cycle"
       - Check if child already exists in this virtual document (409 if duplicate)
       - Calculate next order_index as max(existing order_indexes) + 1, default 0
       - Create VirtualDocumentChild record
       - Audit record (action="add_child")
       - Return the new child record

    5. `async def _detect_cycle(db, virtual_doc_id, candidate_child_doc_id) -> bool`:
       - The virtual document itself has a document_id. If candidate_child_doc_id == virtual_document.document_id, return True (self-reference).
       - Check if candidate_child_doc_id is a virtual document. If so, check if the original virtual_doc's document_id appears as a child (or descendant) of the candidate.
       - Use iterative BFS: start with candidate_child_doc_id, find if it's a virtual doc, get its children, check each child_document_id. If any match the original virtual doc's document_id, cycle found.
       - Continue BFS for any children that are themselves virtual documents.
       - Return True if cycle found, False otherwise.

    6. `async def remove_child(db, virtual_doc_id, child_id, user_id) -> None`:
       - Get virtual document, find the child by id
       - Raise 404 if child not found or doesn't belong to this virtual doc
       - Soft-delete the child record (is_deleted = True)
       - Audit record (action="remove_child")

    7. `async def reorder_children(db, virtual_doc_id, child_ids, user_id) -> list[VirtualDocumentChild]`:
       - Get virtual document with children
       - Validate all provided child_ids match existing non-deleted children (400 if mismatch)
       - Update order_index for each child based on position in child_ids list
       - Flush and return updated children

    8. `async def generate_merged_pdf(db, virtual_doc_id, user_id) -> tuple[bytes, str]`:
       - Get virtual document with children ordered by order_index
       - Raise 400 if no children
       - For each child, find its latest DocumentVersion, then find its PDF rendition (status=READY)
       - Collect all PDF rendition minio_object_keys
       - If any child has no ready PDF rendition, raise 400 with message listing which children lack PDFs
       - Download all PDF bytes from MinIO
       - Use `pypdf.PdfWriter` to merge all PDFs in order
       - Write merged PDF to bytes buffer
       - Return (merged_bytes, filename) where filename is f"{virtual_doc_title}_merged.pdf"
  </action>

    ```python
    from app.celery_app import celery_app

    @celery_app.task(name="app.tasks.virtual_document.merge_pdf_async")
    def merge_pdf_async(virtual_document_id: str, user_id: str) -> dict:
        """Background PDF merge task. Currently the merge is done synchronously
        in the API endpoint since it is fast for typical document counts.
        This task exists for future use with large virtual documents."""
        import asyncio
        from app.core.database import async_session_factory
        from app.services import virtual_document_service
        import uuid

        async def _run():
            async with async_session_factory() as db:
                async with db.begin():
                    merged_bytes, filename = await virtual_document_service.generate_merged_pdf(
                        db, uuid.UUID(virtual_document_id), user_id
                    )
                    # Store in MinIO for later download
                    from app.core.minio_client import upload_object
                    object_key = f"virtual_documents/{virtual_document_id}/merged.pdf"
                    await upload_object(object_key, merged_bytes, "application/pdf")
                    return {"object_key": object_key, "filename": filename, "size": len(merged_bytes)}

        return asyncio.run(_run())
    ```
