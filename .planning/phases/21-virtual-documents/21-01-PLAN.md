# Phase 21 Plan 01: Virtual Document Models, API, and Merged PDF

## Objective

Implement the virtual/compound document feature: SQLAlchemy models for parent-child document assembly with ordering, a service layer with cycle detection and merged PDF generation, Pydantic schemas, and FastAPI endpoints.

## Context

- @src/app/models/document.py - Existing Document/DocumentVersion models
- @src/app/routers/documents.py - Existing document endpoints pattern
- @src/app/schemas/document.py - Existing document schemas
- @src/app/services/document_service.py - Existing document service
- @src/app/main.py - Router registration

## Tasks

### Task 1: Virtual Document Models
type="auto"

Create SQLAlchemy models for VirtualDocument and VirtualDocumentChild.

**Actions:**
- Create `src/app/models/virtual_document.py` with VirtualDocument (extends BaseModel, has title, description, FK to creator) and VirtualDocumentChild (links parent virtual doc to child document with sort_order)
- Register models in `src/app/models/__init__.py`

**Done criteria:** Models defined with proper relationships and constraints.

### Task 2: Schemas and Service
type="auto"

Create Pydantic schemas and service layer with cycle detection.

**Actions:**
- Create `src/app/schemas/virtual_document.py` with create/update/response schemas
- Create `src/app/services/virtual_document_service.py` with CRUD, child management, cycle detection, reorder, and merged PDF generation

**Done criteria:** Service handles all operations including cycle detection and PDF merge.

### Task 3: API Endpoints and Registration
type="auto"

Create FastAPI router and register in main app.

**Actions:**
- Create `src/app/routers/virtual_documents.py` with all CRUD + child management + merge endpoints
- Register router in `src/app/main.py`

**Done criteria:** All endpoints accessible and properly documented.

## Verification

- All new files created and importable
- Router registered in main.py
- Cycle detection prevents circular references

## Success Criteria

1. User can create a virtual document and add existing documents as children in a specified order
2. User can reorder and remove children from virtual documents
3. System detects and prevents circular references
4. User can generate a merged PDF from all children
