- When collapsed: just the icon, centered (`justify-center`)
- Export named `SidebarHeader`

**2. `SidebarUserMenu.tsx`** (per D-08, D-09, D-10, D-14):
- Props: `isCollapsed: boolean`
- Read from authStore: `username`, `isSuperuser`, `isAvailable`, `setAvailability`, `logout`
- Use `useNavigate` for logout redirect to `/login`
- Outer div: `h-14 flex items-center px-3 shrink-0`
- Trigger area (DropdownMenuTrigger): 
  - When expanded: Avatar (32px, initials from first 2 chars of username uppercased as AvatarFallback, bg `--color-muted`) + column of username (14px font-medium truncate, max-w-[140px]) + availability dot (8px circle, inline, `oklch(0.72 0.19 142)` for available, `var(--color-destructive)` for unavailable)
  - When collapsed: Avatar only, centered
- DropdownMenuContent (align="start", side="right" when collapsed, side="bottom" when expanded):
  - DropdownMenuItem (disabled, no hover): username in 14px semibold
  - If isSuperuser: Badge variant="secondary" with text "Admin"
  - DropdownMenuSeparator
  - DropdownMenuItem: availability toggle -- Lucide `Circle` icon (8px, filled with green or red matching dot) + text "Available" or "Unavailable". onClick: call `setAvailability(!isAvailable)`
  - DropdownMenuSeparator
  - DropdownMenuItem: Lucide `LogOut` icon (16px) + "Log out" text. className includes `text-destructive` on hover via `focus:text-destructive`. onClick: call `logout()` then `navigate("/login")`

**3. `SidebarNavItem.tsx`** (per D-07, UI-SPEC Nav Item States table):
- Props: `icon: LucideIcon` (the component type), `label: string`, `route: string`, `isActive: boolean`, `isCollapsed: boolean`
- Use React Router `Link` for navigation (no full page reload)
- Wrap in Tooltip when collapsed: TooltipTrigger around the link, TooltipContent with `label` text (side="right")
- Link className computed with cn():
  - Always: `flex items-center h-10 px-3 rounded-md text-sm transition-colors relative`
  - Active (per D-07): `border-l-[3px] border-primary bg-accent text-foreground` 
  - Not active: `text-muted-foreground hover:bg-accent hover:text-foreground`
  - Collapsed: `justify-center px-0 mx-2`
  - Expanded: `mx-2`
- Icon: rendered at size 20, with `shrink-0`
- Label: `ml-3` (12px gap), hidden when collapsed via `hidden` class
- Export named `SidebarNavItem`

**4. `SidebarNav.tsx`** (per D-06, D-12, UI-SPEC Navigation Items table):
- Props: `isCollapsed: boolean`
- Use `useLocation()` from react-router to get current pathname
- Define nav items array with objects: `{ icon, label, route, adminOnly }`
  - `{ icon: LayoutTemplate, label: "Templates", route: "/templates", adminOnly: false }`
  - `{ icon: Inbox, label: "Inbox", route: "/inbox", adminOnly: false }`
  - `{ icon: FileText, label: "Documents", route: "/documents", adminOnly: false }`
  - `{ icon: GitBranch, label: "Workflows", route: "/workflows", adminOnly: false }`
  - `{ icon: BarChart3, label: "Dashboard", route: "/dashboard", adminOnly: true }`
  - `{ icon: Search, label: "Query", route: "/query", adminOnly: true }`
- Read `isSuperuser` from authStore
- Split into mainItems (adminOnly=false) and adminItems (adminOnly=true)
- Active detection: `pathname.startsWith(item.route)` for each item
- Render structure:
  - If not collapsed: label "Main" in 12px text-muted-foreground uppercase tracking-wider, `px-4 py-2`
  - Map mainItems to SidebarNavItem
  - If isSuperuser: Separator + (if not collapsed: label "Admin") + map adminItems to SidebarNavItem
- Wrap entire nav in TooltipProvider (delayDuration=0) so tooltips work on collapsed items
- Export named `SidebarNav`

**5. `SidebarToggle.tsx`** (per UI-SPEC Sidebar Toggle):
- Props: `isCollapsed: boolean`, `onToggle: () => void`
- Render Button variant="ghost" size="sm" with `h-10 w-full flex items-center justify-center`
- Icon: `ChevronLeft` when expanded, `ChevronRight` when collapsed (20px)
- Wrap in Tooltip: content "Collapse sidebar" when expanded, "Expand sidebar" when collapsed (side="right")
- Export named `SidebarToggle`

**6. `Sidebar.tsx`** (per D-01, D-02, D-03, UI-SPEC Layout Specification):
- Manages collapsed state: `const [isCollapsed, setIsCollapsed] = useState(() => localStorage.getItem('sidebar-collapsed') !== 'false')` (default collapsed per D-02)
- Manages hover-to-peek state: `const [isPeeking, setIsPeeking] = useState(false)`
- Manages mobile open state: `const [isMobileOpen, setIsMobileOpen] = useState(false)`
- On toggle: flip `isCollapsed`, persist to `localStorage.setItem('sidebar-collapsed', String(newValue))`
- Compute effective expanded: `!isCollapsed || isPeeking`
- Compute width: effectiveExpanded ? 240 : 56 (px)
- Hover handlers (desktop only, not when mobile):
  - onMouseEnter: if isCollapsed, start 150ms timeout then setIsPeeking(true)
  - onMouseLeave: clear enter timeout, start 300ms timeout then setIsPeeking(false)
  - Clean up timeouts in useEffect return
- Desktop sidebar (hidden below 768px via `hidden md:flex`):
  - Position: `fixed left-0 top-0 h-screen z-40 flex flex-col bg-secondary border-r border-border`
  - Width transition: `transition-all duration-200 ease-in-out`, style={{ width: `${width}px` }}
  - If isPeeking: add `shadow-[4px_0_12px_rgba(0,0,0,0.08)]`
  - Children order: SidebarHeader, SidebarUserMenu, Separator, SidebarNav, flex-1 spacer, SidebarToggle
  - Pass `isCollapsed={!effectiveExpanded}` to all children
- Mobile (visible below 768px via `md:hidden`):
  - Top bar: `fixed top-0 left-0 right-0 h-12 bg-secondary border-b z-40 flex items-center px-4`
    - Hamburger button: Lucide `Menu` icon (24px), onClick toggles isMobileOpen
    - "Documentum" text centered (flex-1 text-center)
  - Overlay: if isMobileOpen, render backdrop (`fixed inset-0 bg-black/20 z-40`) + sidebar (`fixed left-0 top-0 h-screen w-60 bg-secondary z-50 flex flex-col`)
    - Clicking backdrop or any nav item closes mobile sidebar (setIsMobileOpen(false))
    - Pass `isCollapsed={false}` to all children (always expanded in mobile overlay)
- Export named `Sidebar`
- Also export a context or prop for mainContentMargin: the sidebar provides its current width so AppShell can set margin-left. Use a simple approach: Sidebar renders children with the sidebar + main area together. Accept `children: React.ReactNode` and render:
  - Desktop main area: `<main>` with inline `style={{ marginLeft: width + 'px' }}` and `className="transition-all duration-200 ease-in-out hidden md:block"`. Children inside get the page content.
  - Mobile main area: `<main className="pt-12 md:hidden">` for top-bar clearance.
  - Use `window.matchMedia('(max-width: 767px)')` in a useEffect with resize listener to track `isMobile` state, and conditionally render the appropriate main wrapper, OR simpler: render both desktop and mobile main areas with responsive visibility classes (`hidden md:block` and `md:hidden`).
