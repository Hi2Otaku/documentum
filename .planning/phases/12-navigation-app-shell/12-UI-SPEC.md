## Interaction Contracts

### Sidebar Toggle (D-01, D-02, D-03)

1. User clicks toggle button at sidebar bottom -> sidebar collapses/expands with 200ms transition
2. Preference stored in `localStorage` key `sidebar-collapsed` (boolean)
3. Default state on first load: collapsed (D-02)
4. When collapsed, hovering over sidebar for 150ms triggers peek (overlay expand). Moving mouse out triggers 300ms collapse delay to prevent flicker.

### User Menu (D-08, D-09, D-10)

1. Click on user area opens Radix DropdownMenu
2. Menu contents (top to bottom):
   - Username (non-interactive, 14px semibold)
   - Admin badge if `is_superuser` (shadcn Badge, variant "secondary", text "Admin")
   - Separator
   - Availability toggle: "Available" / "Unavailable" with status dot, click toggles and calls `PATCH /api/users/{id}/availability`
   - Separator
   - "Log out" button (text destructive on hover)
3. When collapsed sidebar: user area shows Avatar with initials only. Click still opens dropdown.

### Role-Based Visibility (D-12, D-13)

1. Sidebar reads `is_superuser` from authStore
2. Admin nav section (Dashboard, Query) rendered only when `is_superuser === true`
3. `AdminRoute` component wraps admin routes: if `!is_superuser`, navigate to `/inbox` with `replace` (D-13)

### Active State Detection

1. Use `useLocation()` from React Router
2. Match logic: `pathname.startsWith(item.route)` for nested routes (e.g., `/templates/123/edit` highlights "Templates")
3. Only one nav item active at a time

## State Management Additions

The `authStore` (Zustand) must be extended with these fields:

```typescript
interface AuthState {
  token: string | null;
  isAuthenticated: boolean;
  username: string | null;
  isSuperuser: boolean;
  isAvailable: boolean;
  setToken: (token: string) => void;
  setAvailability: (available: boolean) => void;
  logout: () => void;
}
```

- `username`, `isSuperuser`: decoded from JWT payload on `setToken` (no additional API call needed)
- `isAvailable`: fetched from `GET /api/users/me` on app load, updated optimistically on toggle

Sidebar collapsed state stored separately in `localStorage` (not Zustand) -- it is a layout preference, not application state.
