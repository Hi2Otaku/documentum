  <action>
1. Create `frontend/src/api/users.ts` with two exported functions:
   - `fetchUserProfile(token: string, userId: string): Promise<UserResponse>` -- calls `GET /api/v1/users/${userId}` with Authorization Bearer header, returns `json.data` (EnvelopeResponse unwrap)
   - `updateAvailability(token: string, isAvailable: boolean, delegateId?: string): Promise<UserResponse>` -- calls `PUT /api/v1/users/me/availability` with JSON body `{ is_available, delegate_id }` and Authorization Bearer header, returns `json.data`
   - Define a TypeScript interface `UserProfile` with fields: `id: string, username: string, email: string | null, is_superuser: boolean, is_available: boolean, delegate_id: string | null`

2. Rewrite `frontend/src/stores/authStore.ts` per D-11 and the UI-SPEC state management section:
   - Interface `AuthState` with fields: `token: string | null`, `isAuthenticated: boolean`, `userId: string | null`, `username: string | null`, `isSuperuser: boolean`, `isAvailable: boolean`, `setToken: (token: string) => void`, `loadProfile: () => Promise<void>`, `setAvailability: (available: boolean) => void`, `logout: () => void`
   - `setToken`: store token in localStorage, decode JWT payload via `JSON.parse(atob(token.split('.')[1]))` to extract `sub` (as userId) and `username`, set `isAuthenticated: true`
   - `loadProfile`: read token and userId from state, call `fetchUserProfile(token, userId)`, set `isSuperuser` and `isAvailable` from response
   - `setAvailability`: optimistically set `isAvailable` in state, then call `updateAvailability(token, available)`. On error, revert. (Note: for now, pass `delegateId` as undefined -- delegation UI is Phase 13)
   - `logout`: clear localStorage token, reset all fields to defaults (`token: null, isAuthenticated: false, userId: null, username: null, isSuperuser: false, isAvailable: true`)
   - On store creation, if token exists in localStorage, decode JWT to set userId and username immediately (same atob decode as setToken)

3. Update `frontend/src/pages/LoginPage.tsx`:
   - After `setToken(token)`, call `await useAuthStore.getState().loadProfile()` before navigating
   - Change the redirect target from `/templates` to `/inbox` (the new default landing page per standard Documentum workflow -- inbox is the daily-use surface)
   - Update the `isAuthenticated` redirect to also go to `/inbox`
