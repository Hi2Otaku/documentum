    1. Create `frontend/src/hooks/useSaveTemplate.ts` (per D-13, DESIGN-07):
       - Export `useSaveTemplate(templateId: string)` hook returning `{ save, validateAndInstall, saving, validating, validationErrors }`
       - Internal state: `saving: boolean`, `validating: boolean`, `validationErrors: ValidationErrorDetail[]`, `lastSavedSnapshot: { nodes: Node[], edges: Edge[] } | null`

       **save() function** (per D-13, UI-SPEC Section 7 "Save Flow"):
       - Strategy: incremental diff between current canvas state and last-saved snapshot
       - Compare current nodes vs lastSavedSnapshot.nodes:
         - Nodes with `data.apiId` that don't exist in lastSaved -> should not happen (they came from API)
         - Nodes WITHOUT `data.apiId` -> these are new: call `createActivity(templateId, activityData)` for each, then store the returned API id back into `node.data.apiId`
         - Nodes WITH `data.apiId` where data changed -> call `updateActivity(templateId, apiId, updates)` for each. Include `position_x: node.position.x`, `position_y: node.position.y` in updates.
         - Nodes in lastSaved but missing from current -> call `deleteActivity(templateId, apiId)` for each
       - Same pattern for edges: new edges (no apiId) -> `createFlow`, changed -> `updateFlow`, removed -> `deleteFlow`
       - IMPORTANT: Create activities BEFORE flows (flows reference activity IDs). When creating new activities, update the node's apiId, then when creating flows for new edges, use the source/target node's updated apiId.
       - After all API calls succeed: update `lastSavedSnapshot` to current state, call `useDesignerStore.getState().setClean()`, show `toast.success('Template saved')`
       - On failure: show `toast.error('Failed to save template. Check your connection and try again.')` with action button that retries
       - For the first load (lastSavedSnapshot is null), populate it from the initially loaded template data

       **Simpler alternative if diff is too complex:** Delete all existing activities/flows, then POST all current ones fresh. This is less efficient but simpler and avoids diff bugs. The trade-off is more API calls but guaranteed consistency. Use this approach if the incremental approach would exceed ~80 lines of code.

       **validateAndInstall() function** (per D-16, DESIGN-06, UI-SPEC Section 7 "Validate & Install Flow"):
       - First: call `save()` to persist current state
       - Then: call `validateTemplate(templateId)` API
       - If `result.valid === true`:
         - Call `installTemplate(templateId)`
         - Show `toast.success('Template installed successfully')`
         - Clear validationErrors
       - If `result.valid === false`:
         - Set `validationErrors` to `result.errors`
         - Show `toast.error('{N} validation errors found')`
         - Call `useUiStore.getState().setErrorPanelExpanded(true)` to open error panel
         - Map each error's `entity_id` to a canvas node and add visual indicator:
           - For each error, find node where `node.data.apiId === error.entity_id` or `node.id === error.entity_id`
           - Update those nodes with a `hasError: true` flag in their data (will need to handle this in node components via a conditional `ring-destructive` class)

    2. Create `frontend/src/hooks/useKeyboardShortcuts.ts` (per D-04, D-05, D-06, UI-SPEC Section 7):
       - Export `useKeyboardShortcuts()` hook
       - Uses `useEffect` to add `keydown` listener on `document`
       - Shortcuts:
         - `Ctrl+S` / `Cmd+S`: prevent default, call save callback (passed as parameter or from context)
         - `Ctrl+Z` / `Cmd+Z`: prevent default, call `useDesignerStore.getState().undo()`
         - `Ctrl+Y` / `Cmd+Y` OR `Ctrl+Shift+Z` / `Cmd+Shift+Z`: prevent default, call `useDesignerStore.getState().redo()`
         - `Delete` / `Backspace`: get selected elements from store, call `deleteElements(selectedNodeIds, selectedEdgeIds)`. Only when focus is NOT on an input/textarea (check `document.activeElement?.tagName`)
         - `Ctrl+A` / `Cmd+A`: prevent default, select all nodes and edges in the React Flow instance
         - `Escape`: call `clearSelection()`
       - Clean up listener on unmount
       - Accept `onSave` callback as parameter to wire to save hook

    3. Create `frontend/src/components/designer/ContextMenu.tsx` (per D-05, UI-SPEC Section 7):
       - Use shadcn `<DropdownMenu>` component
       - Props: `position: { x: number, y: number } | null`, `target: { type: 'node' | 'edge' | 'pane', id?: string } | null`, `onClose: () => void`
       - Render as absolutely positioned at `position` when not null
       - Menu items based on target type (per UI-SPEC):
         - Node: "Delete" (red text, Trash icon) -> calls `deleteElements([target.id], [])`
         - Edge: "Delete" (red text, Trash icon) -> calls `deleteElements([], [target.id])`
         - Pane: "Select All" -> select all, "Auto-Layout" -> run getLayoutedElements, "Fit View" -> call `useReactFlow().fitView()`
       - Close on item click or click outside
       - Wire `onContextMenu` in Canvas.tsx to show this menu (will need to integrate in DesignerPage)
