# Phase 08 Plan 01: React Frontend Setup & Workflow Canvas

## Objective

Set up the React + TypeScript + Vite frontend project with React Flow (@xyflow/react) integration, custom activity node types (Start, End, Manual, Auto), custom edge types (Normal Flow, Reject Flow), and the core workflow designer canvas with drag-and-drop from a palette. Wire save/load to the backend template API.

## Context

- Backend API already supports full template CRUD at `/api/v1/templates/`
- Activity types: START, END, MANUAL, AUTO (from backend enums)
- Flow types: NORMAL, REJECT
- Backend stores position_x/position_y on ActivityTemplate
- CORS already configured with allow_origins=["*"]
- Tech stack: React 19, TypeScript 5, Vite 6, @xyflow/react 12.10.x, Tailwind CSS 4, shadcn/ui, Zustand 5, TanStack Query 5, React Router 7

## Tasks

### Task 1: Initialize React + Vite project with dependencies
type="auto"

**Do:**
- Create `frontend/` directory at project root
- Initialize Vite React TypeScript project
- Install core dependencies: @xyflow/react, tailwindcss, @tanstack/react-query, zustand, react-router
- Configure Vite proxy to forward `/api` requests to `http://localhost:8000`
- Set up Tailwind CSS
- Create basic app shell with React Router (routes: /designer/:id, /templates)
- Verify project builds successfully

**Done when:** `npm run build` succeeds with no errors.

### Task 2: Custom node types and edge types for React Flow
type="auto"

**Do:**
- Create custom React Flow node components:
  - StartNode: green circle/rounded node
  - EndNode: red circle/rounded node
  - ManualNode: blue rectangular node with performer info
  - AutoNode: purple rectangular node with method info
- Create custom edge types:
  - NormalEdge: solid line with arrow
  - RejectEdge: dashed red line with arrow
- Create a Zustand store for designer state (selected node, panel visibility)
- Create TypeScript types mapping backend schema to React Flow nodes/edges

**Done when:** All node and edge components are defined with proper TypeScript types.

### Task 3: Workflow designer canvas with drag-and-drop palette
type="auto"

**Do:**
- Create WorkflowDesigner page component with React Flow canvas
- Create a sidebar palette with draggable activity types (Start, End, Manual, Auto)
- Implement drag-and-drop from palette onto canvas (onDrop handler creates new nodes)
- Implement edge creation by dragging between node handles
- Create API client module for template CRUD operations
- Implement save: convert React Flow nodes/edges to backend API format and POST/PUT
- Implement load: fetch template detail from API, convert to React Flow nodes/edges
- Add toolbar with Save, Validate, and Install buttons
- Wire Validate and Install buttons to backend endpoints

**Done when:** User can create a template visually, add nodes via drag-and-drop, connect them with edges, save to backend, and reload from backend.

## Verification

- `npm run build` succeeds
- All TypeScript types compile without errors
- The designer page renders React Flow canvas with custom nodes
- Drag-and-drop from palette creates new activity nodes on the canvas
- Save/load round-trips data through the backend API

## Success Criteria

- React frontend project initialized with all required dependencies
- Custom node types render correctly for all 4 activity types
- Custom edge types render for Normal and Reject flows
- Drag-and-drop palette creates new nodes on canvas
- Template data can be saved to and loaded from the backend API
