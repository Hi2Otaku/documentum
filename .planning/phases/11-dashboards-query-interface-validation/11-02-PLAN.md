    2. **Create src/app/services/query_service.py** with async query functions following the audit.py pattern:

       - `async def query_workflows(db, template_id=None, state=None, date_from=None, date_to=None, started_by=None, skip=0, limit=20)`:
         Build conditions list. Join ProcessTemplate for template_name. Join User (via supervisor_id) for started_by username. Subquery for active_activity: select ActivityInstance where state=ACTIVE joined with ActivityTemplate for name. Return list[WorkflowQueryResponse] + total_count.

       - `async def query_work_items(db, assignee_id=None, state=None, workflow_id=None, priority=None, skip=0, limit=20)`:
         Build conditions. If workflow_id: join ActivityInstance to filter by workflow_instance_id. Join ActivityTemplate for activity_name. Join WorkflowInstance -> ProcessTemplate for workflow_name. Join User for assignee username. Return list[WorkItemQueryResponse] + total_count.

       - `async def query_documents(db, lifecycle_state=None, metadata_key=None, metadata_value=None, version=None, skip=0, limit=20)`:
         Build conditions. lifecycle_state filter: `Document.lifecycle_state == lifecycle_state`. metadata filter: For top-level key-value matching per RESEARCH.md recommendation, use JSON containment. For SQLite compatibility, fetch all and filter in Python OR use `Document.custom_properties[key].as_string() == value` -- safest approach is to do a LIKE on cast-to-string for the JSON field, but simpler: just filter lifecycle_state and version in SQL, then filter metadata in Python post-fetch if metadata_key is provided. version filter: compare `f"{Document.current_major_version}.{Document.current_minor_version}"` -- since this is two columns, compute in Python after fetch. Return list[DocumentQueryResponse] + total_count.

       Each function returns a tuple: `(list[ResponseModel], total_count: int)`.
