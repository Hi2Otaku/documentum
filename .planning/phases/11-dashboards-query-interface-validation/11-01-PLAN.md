# Phase 11 Plan 01: Dashboard Service, Query Interface & API Endpoints

## Objective

Build the BAM dashboard service and DQL-like admin query interface with API endpoints for real-time workflow metrics, bottleneck detection, workload analysis, and multi-criteria search across workflows, work items, audit logs, and documents.

## Context

- @src/app/models/workflow.py — WorkflowInstance, ActivityInstance, WorkItem models
- @src/app/models/audit.py — AuditLog model
- @src/app/models/document.py — Document model
- @src/app/services/engine_service.py — Existing engine service pattern
- @src/app/schemas/common.py — EnvelopeResponse, PaginationMeta

## Tasks

### Task 1: Dashboard schemas & service
type="auto"

Create Pydantic schemas for dashboard responses and a dashboard_service module that queries workflow metrics.

**Done criteria:**
- `src/app/schemas/dashboard.py` with response models for workflow counts, bottleneck analysis, user workload, and template metrics
- `src/app/services/dashboard_service.py` with functions: `get_workflow_summary()`, `get_bottleneck_activities()`, `get_user_workload()`, `get_template_metrics()`

### Task 2: Query schemas & service
type="auto"

Create a query service that supports multi-criteria filtering of workflows, work items, documents, and audit logs.

**Done criteria:**
- `src/app/schemas/query.py` with request/response schemas for query filters
- `src/app/services/query_service.py` with functions: `query_workflows()`, `query_work_items()`, `query_documents()`, `query_audit_logs()`

### Task 3: Dashboard & query API router + registration
type="auto"

Create HTTP endpoints for dashboard data and query interface, register in main.py.

**Done criteria:**
- `src/app/routers/dashboard.py` with GET endpoints for all dashboard metrics
- `src/app/routers/query.py` with POST endpoints for multi-criteria queries
- Both routers registered in `main.py`
- All endpoints require authentication

### Task 4: Tests for dashboard and query endpoints
type="auto"

Write integration tests covering all dashboard and query endpoints.

**Done criteria:**
- `tests/test_dashboard.py` with tests for workflow summary, bottleneck detection, user workload, template metrics
- `tests/test_query.py` with tests for all query filters (state, date range, performer, template)
- All tests pass
