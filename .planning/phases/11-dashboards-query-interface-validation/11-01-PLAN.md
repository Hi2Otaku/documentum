    4. **Create src/app/services/dashboard_service.py** with async functions:
       - `get_kpi_metrics(db, template_id=None) -> KpiMetrics`: Query WorkflowInstance grouped by state with COUNT. For avg_completion_hours: compute difference between completed_at and started_at for FINISHED workflows. Use Python-side calculation (fetch timestamps, compute timedelta) to avoid SQLite/PostgreSQL dialect differences per RESEARCH.md Pitfall 1.
       - `get_bottleneck_data(db, template_id=None) -> list[BottleneckActivity]`: Query ActivityInstance joined with ActivityTemplate. For COMPLETE activities, compute avg(completed_at - started_at) in Python. Group by activity_template_id, sort descending, limit 10.
       - `get_workload_data(db, template_id=None) -> list[UserWorkload]`: Query WorkItem grouped by performer_id. Count per state: AVAILABLE/ACQUIRED = assigned, COMPLETE = completed, AVAILABLE = pending. Join User to get username.
       - `get_sla_data(db, template_id=None) -> list[SlaCompliance]`: Query WorkItem joined with ActivityInstance -> ActivityTemplate WHERE expected_duration_hours IS NOT NULL. For each activity, count work items completed within the time limit (on_time) vs those exceeding it (overdue). Compute compliance_percent = on_time / (on_time + overdue) * 100.
       - `get_all_metrics(db, template_id=None) -> DashboardMetrics`: Calls all four above and returns combined DashboardMetrics.

    5. **Create src/app/routers/dashboard.py**:
       - `router = APIRouter(prefix="/dashboard", tags=["dashboard"])`
       - `GET /metrics` — calls dashboard_service.get_all_metrics, returns EnvelopeResponse[DashboardMetrics]. Accepts optional `template_id: uuid.UUID | None = Query(None)`. Requires `get_current_active_admin`.
       - `GET /stream` — SSE endpoint using `EventSourceResponse` from `fastapi.sse`. Accepts `token: str = Query(...)` and optional `template_id: uuid.UUID | None = Query(None)`. Validates token manually by decoding JWT (use `decode_access_token` from `app.core.security`, then query User from DB). Returns AsyncIterable[ServerSentEvent]. Loop: create new session via `async_session_factory()` each iteration, call `get_kpi_metrics`, yield `ServerSentEvent(data=json.dumps(kpi.model_dump()), event="kpi_update")`, sleep 5 seconds. Per RESEARCH.md Pitfall 2: do NOT use Depends(get_db) for SSE — create session inside loop.
       - Per RESEARCH.md Pitfall 3: SSE token passed as query param because browser EventSource API cannot send headers.
