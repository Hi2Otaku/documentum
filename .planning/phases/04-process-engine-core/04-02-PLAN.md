**Section 2: Variable resolution helper**

```python
def _resolve_variable_value(pv: ProcessVariable) -> Any:
    """Extract typed value from ProcessVariable based on variable_type."""
    match pv.variable_type:
        case "string": return pv.string_value
        case "int": return pv.int_value
        case "boolean": return pv.bool_value
        case "date": return pv.date_value
        case _: return pv.string_value

def _build_variable_context(variables: list[ProcessVariable]) -> dict[str, Any]:
    return {pv.name: _resolve_variable_value(pv) for pv in variables}
```

**Section 3: `start_workflow` function (per D-06: immediate start)**

Signature: `async def start_workflow(db: AsyncSession, template_id: uuid.UUID, user_id: str, document_ids: list[uuid.UUID] | None = None, performer_overrides: dict[str, str] | None = None, initial_variables: dict[str, Any] | None = None) -> WorkflowInstance`

Steps:
1. Load installed ProcessTemplate with `selectinload` for activity_templates, flow_templates, process_variables. Raise ValueError("Template not found or not installed") if not found or `is_installed is False`.
2. Create WorkflowInstance with `state=WorkflowState.RUNNING`, `started_at=datetime.now(timezone.utc)`, `supervisor_id=uuid.UUID(user_id)`, `created_by=user_id`.
3. `await db.flush()` to get instance.id.
4. Create ActivityInstance for each ActivityTemplate. Build `template_to_instance: dict[uuid.UUID, ActivityInstance]` mapping template ID to instance. All start as `ActivityState.DORMANT`.
5. Copy ProcessVariable from template to instance — set `workflow_instance_id=instance.id`, `process_template_id=None`. Apply `initial_variables` overrides by matching variable name and setting the correct typed column.
6. Create WorkflowPackage for each document_id in `document_ids` (optional per D-04).
7. `await db.flush()` to get all IDs.
8. Find start ActivityTemplate (activity_type == ActivityType.START). Raise ValueError("Template has no start activity") if missing.
9. Call `await _advance_from_activity(db, instance, template_to_instance[start_template.id], template, template_to_instance, user_id)` — the start activity is activated and immediately completed, then the loop advances.
10. Create audit record: `entity_type="workflow_instance"`, `action="workflow_started"`.
11. Return instance.

**Section 4: `_advance_from_activity` — iterative loop (per D-03)**

Signature: `async def _advance_from_activity(db: AsyncSession, workflow: WorkflowInstance, completed_activity: ActivityInstance, template: ProcessTemplate, template_to_instance: dict[uuid.UUID, ActivityInstance], user_id: str) -> None`

Implementation (per D-03, iterative, no recursion):
1. Mark `completed_activity.state = ActivityState.COMPLETE`, set `completed_at`.
2. Initialize `queue = [completed_activity]`.
3. While queue:
   a. Pop from front: `current = queue.pop(0)`.
   b. Get outgoing NORMAL flows from `template.flow_templates` where `source_activity_id == current.activity_template_id` and `flow_type == FlowType.NORMAL`.
   c. For each outgoing flow:
      - If flow has `condition_expression`, evaluate it against `_build_variable_context(workflow.process_variables)` using `evaluate_expression`. Skip flow if result is False.
      - Place token: create `ExecutionToken(workflow_instance_id=workflow.id, flow_template_id=flow.id, source_activity_instance_id=current.id, target_activity_template_id=flow.target_activity_id, is_consumed=False)`. Add to db.
      - Check if target should activate via `_should_activate()`.
      - If should activate:
        - **CRITICAL: OR-join double-activation guard** — Before activating, check `target.state == ActivityState.DORMANT`. If the target is already ACTIVE or COMPLETE (e.g., OR-join already fired from a different branch), skip activation entirely. This prevents creating duplicate work items or re-completing activities. Log a debug message when skipping: `f"Skipping activation of {target.activity_template_id}: already {target.state.value}"`.
        - Get target ActivityInstance from `template_to_instance[flow.target_activity_id]`.
        - Mark tokens consumed: set `is_consumed=True` on all unconsumed tokens for this workflow + target activity template.
        - Activate: set target `state=ActivityState.ACTIVE`, `started_at=now()`.
        - If target is START or END type: immediately complete it (set COMPLETE + completed_at) and add to queue.
        - If target is MANUAL type: create a WorkItem with `performer_id` from template's `performer_id` (or from `performer_overrides` if provided), `state=WorkItemState.AVAILABLE`.
   d. After processing all outgoing flows from current, if no flows fired and current is END type: mark workflow as FINISHED (set `state=WorkflowState.FINISHED`, `completed_at=now()`) per D-08.

**Section 5: `_should_activate` — token check (per D-02)**

Signature: `async def _should_activate(db: AsyncSession, workflow_id: uuid.UUID, target_activity_template_id: uuid.UUID, trigger_type: TriggerType, flow_templates: list[FlowTemplate]) -> bool`

1. Count incoming NORMAL flows for the target activity template from `flow_templates`.
2. Query `ExecutionToken` where `workflow_instance_id == workflow_id` and `target_activity_template_id == target_activity_template_id` and `is_consumed == False`. Count results.
3. If `trigger_type == TriggerType.AND_JOIN`: return `token_count >= len(incoming_flows)` (per EXEC-07).
4. If `trigger_type == TriggerType.OR_JOIN`: return `token_count >= 1` (per EXEC-12).

**Section 6: `complete_work_item` function**

Signature: `async def complete_work_item(db: AsyncSession, workflow_id: uuid.UUID, work_item_id: uuid.UUID, user_id: str, output_variables: dict[str, Any] | None = None) -> WorkItem`

Steps:
1. Load WorkflowInstance. Raise ValueError if not found. Raise ValueError("Workflow is not running") if `state != WorkflowState.RUNNING` (per D-09).
2. Load WorkItem with `selectinload(WorkItem.activity_instance)`. Raise ValueError if not found or already complete.
3. Mark WorkItem as `state=WorkItemState.COMPLETE`, `completed_at=now()`.
4. If `output_variables` provided, update matching ProcessVariable records on the workflow instance.
5. Load the full template with relations (activity_templates, flow_templates, process_variables) and rebuild `template_to_instance` mapping.
6. Call `await _advance_from_activity(db, workflow, work_item.activity_instance, template, template_to_instance, user_id)`.
7. Create audit record: `entity_type="work_item"`, `action="work_item_completed"`.
8. Return work_item.

**Section 7: Query functions**

- `async def get_workflow(db, workflow_id) -> WorkflowInstance` — load with `selectinload` for activity_instances (+ activity_template), process_variables, workflow_packages.
- `async def list_workflows(db, skip=0, limit=20) -> tuple[list[WorkflowInstance], int]` — paginated list, filter `is_deleted=False`.
- `async def get_workflow_work_items(db, workflow_id) -> list[WorkItem]` — all work items for a workflow.
- `async def get_variable(db, workflow_id, variable_name) -> ProcessVariable` — single variable lookup.
- `async def update_variable(db, workflow_id, variable_name, value, user_id) -> ProcessVariable` — update variable value per EXEC-13.

Note: For `performer_overrides` in `start_workflow`, the dict maps `activity_template_id` (as string) to `user_id` (as string). When creating work items, check if the activity template ID is in overrides and use that user_id instead of template's performer_id.

All service functions follow existing pattern: raise ValueError for business rule violations, caller (router) maps to HTTP errors.
