# Phase 22 Plan 02: Retention Policies, Legal Holds & Deletion Blocking

## Objective

Implement complete retention and records management: retention policy CRUD, document-policy assignment, deletion blocking for retained documents, and legal holds that override retention expiration.

## Context

- @src/app/models/document.py -- Document and DocumentVersion models
- @src/app/services/document_service.py -- Document CRUD service
- @src/app/routers/documents.py -- Document API endpoints including delete
- @src/app/models/base.py -- BaseModel with id, timestamps, is_deleted
- @src/app/models/enums.py -- Existing enums
- @src/app/schemas/common.py -- EnvelopeResponse, PaginationMeta

## Tasks

### Task 1: Retention Models & Migration
type="auto"

Create SQLAlchemy models for RetentionPolicy, DocumentRetention, and LegalHold. Create Alembic migration.

**Done criteria:**
- RetentionPolicy model with name, description, retention_period_days, disposition_action (archive/delete)
- DocumentRetention model linking documents to policies with applied_at, expires_at
- LegalHold model with document_id, reason, placed_by, placed_at, released_at
- Alembic migration file created
- Models registered in __init__.py

### Task 2: Schemas & Service Layer
type="auto"

Create Pydantic schemas and the retention_service with full CRUD + assignment + hold logic.

**Done criteria:**
- Pydantic request/response schemas for all three entities
- retention_service with: create/list/get/update/delete retention policies
- assign_policy_to_document and remove_policy_from_document
- place_legal_hold and release_legal_hold
- check_document_deletable function that returns (bool, reason)

### Task 3: API Endpoints & Delete Guard
type="auto"

Create retention router with all endpoints. Hook deletion blocking into the document delete endpoint.

**Done criteria:**
- POST/GET/PUT/DELETE /retention-policies endpoints
- POST/DELETE /documents/{id}/retention endpoint for policy assignment
- POST/DELETE /documents/{id}/legal-hold endpoint
- GET /documents/{id}/retention-status returns combined retention + hold info
- Document delete endpoint returns 403 with clear message when document is under retention or legal hold
- Router registered in main.py

### Task 4: Tests
type="auto"

Write comprehensive tests covering all retention functionality.

**Done criteria:**
- Test retention policy CRUD
- Test document-policy assignment
- Test deletion blocking (retained document cannot be deleted)
- Test legal hold placement and release
- Test legal hold blocks deletion even after retention expires
- All tests pass

## Verification

- All 4 requirements (RET-01 through RET-04) are covered
- Document deletion is blocked when under retention or legal hold
- Legal hold overrides retention expiration
- API follows existing patterns (EnvelopeResponse, admin-only for policy management)

## Success Criteria

1. Admin can create retention policies with period and disposition action
2. Admin can assign policies to documents; assignment visible via API
3. Deletion of retained documents is blocked with clear error
4. Legal holds prevent deletion and override retention expiration
