Add signature immutability checks to two functions in `src/app/services/document_service.py`:

1. **In `checkin_document()`**, after getting the latest version (around line 259 where `latest_version` is fetched), add:
```python
# Block checkin if latest version is signed (SIG-04 immutability)
if latest_version is not None:
    from app.services import signature_service
    has_sigs = await signature_service.check_version_has_signatures(db, latest_version.id)
    if has_sigs:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="Cannot check in: the current version has been digitally signed and is immutable. Create a new version by checking out first.",
        )
```

Note: The check should happen AFTER the lock verification but BEFORE creating the new version. The idea is: if the latest version is signed, we block uploading a replacement. However, since checkin always creates a NEW version (doesn't overwrite), the real immutability concern is about preventing new content on a signed document. Per the requirement "System prevents re-upload, check-in, or metadata modification of a signed document version", we block checkin when the latest version is signed. The user would need to work with the existing signed content.

Actually, re-reading the requirement more carefully: "prevents re-upload, check-in, or metadata modification of a signed document VERSION". This means the signed VERSION is immutable, but the document itself can still get new versions. So the correct behavior is:
- Checkin creates a NEW version, which is fine -- the signed version remains untouched
- The guard should prevent overwriting/replacing a signed version's content, but checkin already creates a new version, so no guard needed on checkin itself
- The guard IS needed on metadata update since it modifies the document (title, author, etc.) -- but metadata is on the Document, not DocumentVersion

Revised approach per requirement semantics:
- **Do NOT block checkin** -- checkin creates a new unsigned version, signed versions remain immutable
- **Block metadata update if any version is signed** -- this protects the document-level metadata. Add this check.
- **Add a helper** `check_document_has_any_signatures(db, document_id)` to signature_service that checks if ANY version of the document has signatures.

2. **In `update_document_metadata()`**, after the `document = await get_document(db, document_id)` line, add:
```python
# Block metadata changes if document has signed versions (SIG-04 immutability)
from app.services import signature_service
has_sigs = await signature_service.check_document_has_any_signatures(db, document_id)
if has_sigs:
    raise HTTPException(
        status_code=status.HTTP_409_CONFLICT,
        detail="Cannot modify metadata: this document has digitally signed versions. Signed documents are immutable.",
    )
```

3. **Add to signature_service.py** (also update this file):
```python
async def check_document_has_any_signatures(db: AsyncSession, document_id: uuid.UUID) -> bool:
    result = await db.execute(
        select(DocumentSignature).where(DocumentSignature.document_id == document_id).limit(1)
    )
    return result.scalar_one_or_none() is not None
```

This means:
- Signed versions cannot be overwritten (checkin creates new versions, so this is inherently safe)
- Document metadata cannot be changed once any version is signed
- The signed version's content in MinIO is never modified
