Create `frontend/src/api/documents.ts` following the exact pattern from `inbox.ts`:

1. Copy the helper functions (authHeaders, apiFetch, apiMutate, buildUrl) from inbox.ts. These are duplicated per module in this codebase -- follow the existing convention, do NOT refactor to a shared module.

2. Define TypeScript interfaces matching the backend schemas:
   - `DocumentResponse` with fields: id (string), title, author (string|null), filename, content_type, custom_properties (Record<string,unknown>), locked_by (string|null), locked_at (string|null), current_major_version (number), current_minor_version (number), current_version (string), created_at (string), updated_at (string), created_by (string|null), is_deleted (boolean), lifecycle_state (string|null)
   - `DocumentVersionResponse` with fields: id (string), document_id (string), major_version (number), minor_version (number), content_hash (string), content_size (number), filename, content_type, comment (string|null), created_at (string), created_by (string|null), version_label (string)
   - `PaginationMeta` with fields: page, page_size, total_count, total_pages (all number)
   - `PaginatedDocumentsResponse` = { data: DocumentResponse[], meta: PaginationMeta }
   - `DocumentListParams` = { page?: number, page_size?: number, title?: string, author?: string }
   - `LifecycleTransitionResponse` = { id: string, lifecycle_state: string|null, title: string, updated_at: string }

3. Implement API functions:
   - `fetchDocuments(params: DocumentListParams)` -> GET `/api/v1/documents` with query params. Returns `PaginatedDocumentsResponse`.
   - `fetchDocument(id: string)` -> GET `/api/v1/documents/{id}`. Unwrap envelope, return `DocumentResponse`.
   - `fetchVersions(documentId: string)` -> GET `/api/v1/documents/{id}/versions`. Unwrap envelope, return `DocumentVersionResponse[]`.
   - `uploadDocument(file: File, title: string, author: string)` -> POST `/api/v1/documents/` as multipart FormData (fields: file, title, author). Use fetch directly with auth headers (no Content-Type header -- let browser set multipart boundary). Unwrap envelope, return `DocumentResponse`.
   - `checkoutDocument(id: string)` -> POST `/api/v1/documents/{id}/checkout`. Unwrap envelope, return `DocumentResponse`.
   - `checkinDocument(id: string, file: File, comment?: string)` -> POST `/api/v1/documents/{id}/checkin` as multipart FormData (fields: file, comment). Unwrap envelope, return `DocumentVersionResponse`.
   - `cancelCheckout(id: string)` -> POST `/api/v1/documents/{id}/unlock`. Unwrap envelope, return `DocumentResponse`. Note: backend requires admin role; the API call will return 403 for non-admin users -- the UI will handle this with a toast error.
   - `transitionLifecycle(id: string, targetState: string)` -> POST `/api/v1/documents/{id}/lifecycle/transition` with JSON body `{ target_state: targetState }`. Unwrap envelope, return `LifecycleTransitionResponse`.
   - `downloadVersionUrl(documentId: string, versionId: string)` -> returns the URL string `/api/v1/documents/{documentId}/versions/{versionId}/download` (no fetch -- used for `<a href>` or window.open).

4. For multipart endpoints (uploadDocument, checkinDocument), do NOT set Content-Type header -- let the browser set it with the boundary. Use `new FormData()`, append fields, then fetch with method POST and the FormData body.
