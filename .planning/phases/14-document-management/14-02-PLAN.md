**DocumentDropZone** (`frontend/src/components/documents/DocumentDropZone.tsx`):
- Props: `{ onUploadComplete: () => void }` -- called after all uploads finish to trigger query invalidation.
- State: `uploadingFiles: Array<{ name: string; status: 'pending' | 'uploading' | 'done' | 'error' }>`, `isDragOver: boolean`.
- Default view (no uploads in progress):
  - Dashed border container: `border-2 border-dashed rounded-lg h-[120px] flex flex-col items-center justify-center gap-2 mb-4 transition-colors`.
  - Background: `bg-secondary`. On drag over: `bg-[oklch(0.97_0.02_250)]` and border changes to `border-primary`.
  - Lucide `Upload` icon (32px, `text-muted-foreground`; on drag: `text-primary`).
  - Text: "Drag files here or click to upload" (14px, muted-foreground). On drag: "Drop files to upload".
  - "Browse" button (variant="outline", size="sm").
  - Hidden `<input type="file" multiple>` ref, triggered by Browse button click AND by clicking the drop zone area.
- Drag events: `onDragOver` (prevent default, set isDragOver), `onDragLeave` (unset isDragOver), `onDrop` (prevent default, unset isDragOver, read `e.dataTransfer.files`, start upload).
- Upload view (files uploading): Drop zone collapses to show list of `UploadProgressItem` rows.
- Upload logic (per D-04, sequential):
  1. Set all files to 'pending' status in uploadingFiles state.
  2. For each file sequentially (for loop, not Promise.all):
     a. Set current file status to 'uploading'.
     b. Call `uploadDocument(file, titleFromFilename, currentUsername)`. Title = filename without extension. Username from authStore.
     c. On success: set status to 'done', show `toast.success("Document uploaded")` via sonner.
     d. On error: set status to 'error', show `toast.error("Upload failed: " + error.message)` via sonner. Continue to next file.
  3. After all files processed, call `onUploadComplete()`.
  4. After 3 seconds (setTimeout), clear uploadingFiles array to return to default view.
- Import `toast` from `sonner` for all success/error notifications.
- Import `useAuthStore` (or read username from localStorage/authStore) for author default.
